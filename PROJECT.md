# Project Context

- `ragtag_crew` 是一个单用户、自部署的 coding agent，当前正式入口是 Telegram 和微信，REPL 作为开发 / 调试入口保留。
- 当前不提前为多前端做复杂抽象。
- 实现目标优先级是：上下文清晰、行为可解释、代码简单可维护。
- 默认尽量保持上下文短，把长内容放到按需读取的层里，而不是每轮全部注入。

## 项目特化实施约束

- 默认选择最小可行实现，不提前把 Telegram、微信、REPL 抽象成统一前端框架。
- 需要跨前端共享时，优先复用必要的公共层，例如 `session_store.py`、`session_routes.py`、`trace.py`；不要为潜在场景提前扩展。
- 修改某一前端或某一命令时，不顺手重写无关前端、上下文层或外部能力层。
- 功能改动优先补 `tests/` 中的行为测试，确保命令、会话路由和上下文注入可以直接验证。
